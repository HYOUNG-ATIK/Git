## Git

= **버전 관리 시스템(VCS)**

><code>하향식 학습</code> : 명령어부터 암기 후 사용 => 유연한 사고 퇴화 <br>
<code>좋은 디자인</code> : 암기하지 않아도 이해가능 <br>
>>**개발 목표** <br>
>><code>빠른 속도</code>&nbsp;<code>단순한 구조</code>&nbsp;<code>비선형적 개발</code>
&nbsp;<code>분산</code>&nbsp;<code>대형 프로젝트에 유용</code>



### :pushpin: CVCS vs. DVCS

> ==로컬 VCS -> 중앙집중 VCS -> 분산 VCS== <br>
>> 비교|CVCS|DVCS
 -|-|-
 방식|중앙집중식버전관리시스템|분산버전관리시스템
 예시||***git***
 장점|관리자가 관리하기 쉬움|저장소를 히스토리와 함께 전부 복제, 서버에 문제가 발생해도 복원 가능
 단점|중앙 DB에 문제가 생기면 모든 히스토리 손실|

### :pushpin: 다른 VCS와 비교

Git의 가장 큰 차이점은 __*"데이터를 다루는 방법"*__


### :pushpin: 데이터 모델

데이터 모델 이해 후 모델 컨트롤 측면에서 명령어 이해 = <code>상향식 학습</code>

> **Staging Area** 
<br>
데이터 모델과는 또 다른 개념이지만 커밋을 생성하는 인터페이스의 일부
